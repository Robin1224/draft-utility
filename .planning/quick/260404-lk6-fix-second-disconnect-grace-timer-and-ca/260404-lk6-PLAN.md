<objective>
Fix two post-UAT bugs in the disconnect-resilience flow.

Bug 1: After a captain reconnects and disconnects a second time, the grace timer in onUnsubscribe
is never scheduled because the code path only reads ds.paused — which is false after resumption —
but does not account for the previous disconnect having already consumed the promotion opportunity.
The fix tracks reconnect count in draftState and applies a shorter 10s cancel-only grace window on
second and subsequent disconnects.

Bug 2: The cancelledTeam derived value in DraftBoard.svelte scans members for isCaptain. After
a draft is cancelled the captain row still exists in the snapshot (the member was not removed, only
the draft was cancelled), so both teams appear to have a captain and the fallback returns 'Unknown'.
The fix reads the team directly from the script at turnIndex — the turn that was active when the
draft was cancelled.

Purpose: Correct disconnection edge-case behaviour and the cancelled draft banner label.
Output: Updated room.js and DraftBoard.svelte.
</objective>

async function disconnectGraceExpiredCancelOnly(publicCode, publish) {
  const code = normalizePublicCode(publicCode);
  const roomRow = await getRoomByPublicCode(db, code);
  if (!roomRow || roomRow.phase !== 'drafting') return;
  const ds = typeof roomRow.draft_state === 'string'

    : roomRow.draft_state;

  const snapBeforeCancel = await loadDraftSnapshot(db, code);
  await cancelDraftNoCaption(db, roomRow.id);

  <name>Task 2: Fix cancelledTeam derived in DraftBoard.svelte to use script turnIndex</name>
  <files>src/lib/components/molecules/DraftBoard.svelte</files>
  <action>
Replace the cancelledTeam IIFE in DraftBoard.svelte.

Current code (lines 76-84):
```js
const cancelledTeam = $derived((() => {
  if (snapshot.phase !== 'cancelled') return null;
  for (const t of ['A', 'B']) {
    const hasCaptain = (snapshot.teams[t] ?? []).some((m) => m.isCaptain);
    if (!hasCaptain) return t;
  }
  return 'Unknown';
})());
```

Replace with:
```js
const cancelledTeam = $derived((() => {
  if (snapshot.phase !== 'cancelled') return null;
  // Use the script turn that was active at cancellation — the team that lost its captain.
  // Scanning isCaptain is unreliable: the member row remains in the snapshot after cancellation.
  const ds = snapshot.draftState;
  if (ds?.script && ds.turnIndex != null) {
    return ds.script[ds.turnIndex]?.team ?? 'Unknown';
  }
  return 'Unknown';
})());
```

Also update the PauseOverlay timerMs prop: the overlay currently hardcodes 30000 but must
reflect the actual grace window. Pass graceMs derived from draftState:

```svelte
{#if snapshot.draftState?.paused && snapshot.phase !== 'cancelled'}
  <PauseOverlay
    captainName={pausedCaptainName}
    graceEndsAt={snapshot.draftState.graceEndsAt ?? ''}
    timerMs={snapshot.draftState.graceEndsAt
      ? new Date(snapshot.draftState.graceEndsAt).getTime() - Date.now() + (snapshot.draftState.timerMs ?? 30000)
      : 30000}
  />
{/if}
```

Wait — PauseOverlay's timerMs is used as the total countdown duration to compute the progress
ring, not as remaining time. The simplest fix: derive the initial grace window from the stored
graceEndsAt vs now at mount time. But PauseOverlay is a separate component — changing its API
is out of scope for this quick fix.

Simpler correct approach: pass the initial grace duration as the difference between graceEndsAt
and (graceEndsAt minus graceMs). We don't know graceMs at the client directly. Instead, keep
timerMs={30000} for first disconnect and add a derived that checks reconnectCount:

```svelte
{#if snapshot.draftState?.paused && snapshot.phase !== 'cancelled'}
  <PauseOverlay
    captainName={pausedCaptainName}
    graceEndsAt={snapshot.draftState.graceEndsAt ?? ''}
    timerMs={(snapshot.draftState.reconnectCount ?? 0) >= 1 ? 10_000 : 30_000}
  />
{/if}
```

This keeps the PauseOverlay countdown accurate for both first and second disconnects.

Only make the cancelledTeam IIFE change and the PauseOverlay timerMs change. Do not
touch any other parts of the file.
