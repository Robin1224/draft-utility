<objective>
Fix the grace-timer publish gap identified in the v1.0 audit.

Purpose: When a captain disconnects and the grace timer expires on the **final** turn, `autoAdvanceTurn` is called from `disconnectGraceExpired` in `src/live/room.js` with `platform=null`. The current code calls `completeDraft` correctly but the `if (platform)` guard around `loadDraftSnapshot + ctx.publish` means connected clients never receive the reactive push to the review phase. They remain stuck on the draft view until a manual reload.

The fix separates the publish concern from the platform context: after `completeDraft`, publish the review snapshot unconditionally by creating a platform-independent publish path.

Output: Modified `src/live/draft.js` where `autoAdvanceTurn` publishes the review snapshot after `completeDraft` regardless of `platform`, plus a test in `src/live/draft.spec.js` covering the grace-timer completion path.
</objective>

  <action>
Restructure the `autoAdvanceTurn` function in `src/live/draft.js` so that:

1. After `completeDraft` fires (the `isLast` branch), ALWAYS load and publish the snapshot — not guarded by `if (platform)`.

2. For the non-last-turn path (mid-draft advances), keep the existing `if (platform)` guard unchanged — it is correct for the mid-draft case and platform is always present there.

The resulting shape of the `isLast` + publish section should be:

```js
if (isLast) {
    await completeDraft(db, roomRow.id);
    // Always publish review snapshot — platform may be null (grace-timer path)
    // but the publish module-level API does not require a live context.
    try {
        const snap = await loadDraftSnapshot(db, code);
        if (snap) {
            if (platform) {
                platform.publish(topicForRoom(code), 'set', snap);
            } else {
                // Grace-timer path: platform context unavailable, use module-level publish
                const { publish } = await import('svelte-realtime/server');
                publish(topicForRoom(code), 'set', snap);
            }
        }
    } catch {
        // publish failure is non-fatal — clients hydrate on reconnect
    }
} else {
    scheduleTimer(roomRow.id, draftState.timerMs, () => autoAdvanceTurn(code, nextIndex, platform));
}
```

Then remove (or reduce) the existing `if (platform)` block at the bottom of the function that handles mid-draft publishes — keep it only for the `!isLast` branch which already exists there. Since the old block handled both cases with a single guard, after the above change the bottom `if (platform)` block should handle ONLY the non-last case. Remove it from the last-turn path since the last-turn path now publishes inside the `isLast` branch.

IMPORTANT: Verify first that `svelte-realtime/server` exports a module-level `publish(topic, type, data)` function by checking node_modules or existing codebase usage. If `svelte-realtime/server` does NOT export a module-level `publish`, use an alternative: store the `platform` reference from the most recent RPC context in a module-level variable and use it from the grace-timer path. Check `src/live/room.js` for how `disconnectGraceExpired` is structured — if it has access to `ctx.platform`, pass it to `autoAdvanceTurn`.

The simpler fix if module-level publish is unavailable: modify `disconnectGraceExpired` in `src/live/room.js` to pass `ctx.platform` (or the platform stored at subscription time) to `autoAdvanceTurn`. Read `src/live/room.js` first to determine which approach is feasible.
