<objective>
Fix the grace-timer publish gap identified in the v1.0 audit.

Purpose: When a captain disconnects and the grace timer expires on the **final** turn, `autoAdvanceTurn` is called from `disconnectGraceExpired` in `src/live/room.js` with no platform argument. The current code calls `completeDraft` correctly but the `if (platform)` guard around `loadDraftSnapshot + ctx.publish` means connected clients never receive the reactive push to the review phase. They remain stuck on the draft view until a manual reload.

The fix threads the `publish` function that `disconnectGraceExpired` already holds (passed in as its third parameter from `ctx.publish` at the onUnsubscribe call site) into `autoAdvanceTurn` as an optional fourth parameter. No new imports are needed — the publish callable is already available in the call chain.

Output: Modified `src/live/draft.js` where `autoAdvanceTurn` publishes the review snapshot after `completeDraft` regardless of `platform`, plus a test in `src/live/draft.spec.js` covering the grace-timer completion path.
</objective>

  Resolve the callable as `platform?.publish?.bind(platform) ?? publishFn`. If neither is

- For non-last-turn mid-draft advances, keep the existing `if (platform)` guard unchanged —

The resulting shape of the function tail should be:

```js
if (isLast) {
    await completeDraft(db, roomRow.id);
    // Publish review snapshot unconditionally — platform may be null (grace-timer path).
    // publishFn is threaded from disconnectGraceExpired which holds ctx.publish.
    const pub = platform ? platform.publish.bind(platform) : publishFn;
    if (pub) {
        try {
            const snap = await loadDraftSnapshot(db, code);
            if (snap) pub(topicForRoom(code), 'set', snap);
        } catch {
            // publish failure is non-fatal — clients hydrate on reconnect

    scheduleTimer(roomRow.id, draftState.timerMs, () => autoAdvanceTurn(code, nextIndex, platform));

// Mid-draft publish: only when platform is present (non-last-turn path)
if (platform && !isLast) {
    try {

```

Wait — the current code has a single `if (platform)` block at the bottom that covers both
last and non-last turns. After the above restructure the `isLast` branch handles last-turn
publish internally. The bottom block should therefore only fire for the non-last case.
Add `&& !isLast` to the bottom guard, or simply remove the bottom block entirely since the
non-last-turn scheduleTimer call doesn't need an immediate publish (the real pickBan already
published, and the next timer advance will publish when it fires with platform available).

Simplest correct outcome — replace the entire tail section with:

```js
if (isLast) {
    await completeDraft(db, roomRow.id);
    const pub = platform ? platform.publish.bind(platform) : publishFn;
```
