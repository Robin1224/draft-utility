# Phase 3 — UI Design Contract

> Visual and interaction contract for the **Draft Engine** phase: room settings panel (pick/ban script editor + turn timer) within `LobbyHostBar`, and the `Phases` strip transition to "Drafting" when the server advances `room.phase`. Generated by gsd-ui-researcher.
>
> Phase 3 does NOT include pick/ban visuals, countdown rendering, captain turn indicators, or disconnect UI — those are Phase 4 (DRAFT-07, DISC-01–04).

## Component: Settings Panel (new, inside `LobbyHostBar`)

### Trigger

- Button label: **Settings** — `text-sm font-medium text-text-primary`, styled like the Move/Kick controls (neutral border, not accent-colored).
- Toggle behavior: clicking "Settings" shows/hides the inline settings section. Use `$state(false)` boolean — `settingsOpen`.
- Button has `aria-expanded={settingsOpen}` and `aria-controls="draft-settings-panel"`.
- The "Settings" button is only rendered when `room.phase === 'lobby'`. Once the draft starts, the button and panel are removed entirely (not just disabled).

### Panel container

```
id="draft-settings-panel"
class="mt-3 flex flex-col gap-4 rounded-md border border-bg-secondary bg-bg-primary p-4"
```

Panel is hidden when `settingsOpen === false` (use `{#if settingsOpen}` — no CSS-only toggle needed for v1).

### Section label pattern (consistent with existing Kick label)

```
<span class="text-xs font-semibold uppercase text-text-tertiary">Draft settings</span>
```

## Component: Pick/Ban Script Editor (drag-to-reorder)

Inside the settings panel, below the turn timer.

### Structure

```
<span class="text-xs font-semibold uppercase text-text-tertiary">Pick / Ban order</span>
<ul id="script-editor-list" class="mt-2 flex flex-col gap-2" aria-label="Pick/ban turn order">
  {#each script as turn, i (turn.id)}
    <!-- ScriptTurnRow -->
  {/each}
</ul>
<button type="button" class="...">Add turn</button>
```

### ScriptTurnRow

Each row represents one `{ team: 'A' | 'B', action: 'pick' | 'ban' }` turn. Each turn has a stable `id` (e.g. nanoid or index-based) for Svelte keying.

Row layout: `flex items-center gap-2`

| Element | Classes | Notes |
|---------|---------|-------|
| Drag handle | `cursor-grab text-text-tertiary` | Inline SVG grip icon (6-dot or 3-line pattern, 16px); `aria-label="Drag to reorder"` |
| Turn index | `text-sm text-text-tertiary w-5 text-right` | 1-based display only (e.g. "1") |
| Team select | `rounded-md border border-bg-secondary bg-bg-primary px-2 py-1 text-sm text-text-primary` + `w-24` | Options: "Team A", "Team B" |
| Action select | same classes as team select + `w-24` | Options: "Pick", "Ban" |
| Remove button | `text-sm font-medium text-red-600 hover:underline` | Label: "Remove"; `aria-label="Remove turn {i+1}"` |
| Row container | `rounded-md border border-bg-secondary bg-bg-primary px-3 py-2 flex items-center gap-2` | Matches player slot rows in `TeamColumn` |

### "Add turn" button

- Label: **Add turn**
- Classes: `mt-1 rounded-md border border-bg-secondary px-3 py-2 text-sm font-medium text-text-primary hover:bg-bg-secondary` — neutral, not accent. Consistent with Move button in host bar.
- Appends a new `{ team: 'A', action: 'ban' }` turn to the end of the script (default team A, default action ban).

### Drag-to-reorder implementation note

Use the browser-native HTML Drag and Drop API (`draggable="true"`, `ondragstart`, `ondragover`, `ondrop`) on each row. No external DnD library. The drag handle element initiates drag. On drop, reorder the script array in Svelte state. Keyboard-accessible alternative: the Team and Action selects allow manual configuration without drag.

### Default script

The default script loaded when settings panel first opens (D-03 — Claude's discretion: esports-style alternating bans then picks):

```
[
  { team: 'A', action: 'ban' },
  { team: 'B', action: 'ban' },
  { team: 'A', action: 'ban' },
  { team: 'B', action: 'ban' },
  { team: 'A', action: 'pick' },
  { team: 'B', action: 'pick' },
  { team: 'B', action: 'pick' },
  { team: 'A', action: 'pick' },
  { team: 'A', action: 'pick' },
  { team: 'B', action: 'pick' }
]
```

Rationale: 2 bans per team (4 total bans), then 3 picks per team (6 total picks) in snake order — standard competitive esports draft format. Total: 10 turns across 28-champion catalog.

## Phase Strip: Transition to "Drafting"

When the server emits a snapshot with `room.phase === 'drafting'`, the `Phases` atom updates automatically via the reactive store (existing `fromStore(lobby(code)).current` pattern).

- "Lobby" segment: `text-text-tertiary hidden sm:block` (past phase, not active).
- "Drafting" segment: `text-text-secondary underline` (active).
- "Review" segment: `text-text-tertiary hidden sm:block opacity-50 pointer-events-none select-none` (future phase).

No new Svelte code needed in `Phases.svelte` — the existing `phaseToIndex` logic already handles `'drafting'` returning index 2.

**Draft page behavior when `room.phase` transitions to `'drafting'`:**
- `LobbyHostBar` and its settings panel are no longer rendered (the host bar already gates on `isHost` and phase checks; Phase 3 must ensure settings button disappears).
- The main lobby grid (teams, spectators, copy link) remains visible until Phase 4 replaces it with the draft board.
- The `Phases` strip visually advances. That is the extent of the Phase 3 UI change on transition — Phase 4 owns the full drafting view.

## Accessibility Contract

- Settings toggle button: `aria-expanded={settingsOpen}` + `aria-controls="draft-settings-panel"`.
- Settings panel: `id="draft-settings-panel"` for `aria-controls` association.
- Drag handles: `aria-label="Drag to reorder"` on the grip icon button. `role="button"` if using a non-button element.
- Script turn list: `aria-label="Pick/ban turn order"` on the `<ul>`.
- Remove buttons: `aria-label="Remove turn {i+1}"` — not just "Remove".
- Turn timer input: `<label>` element associated via `for`/`id`; `min`, `max`, `step` attributes set for browser validation.
- Team and Action selects: `<label>` associated via `for`/`id` on each row (or `aria-label` if space is tight).
- Keyboard drag alternative: selects allow configuration without drag; no additional keyboard DnD required for v1.
