> WebSocket transport layer (hooks.ws.js) has no user-facing UI and is excluded from this contract.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, tight inline spacing |
| sm | 8px | `p-2`, `gap-2` | Button vertical padding, compact gaps |
| md | 16px | `gap-4`, `p-4` | Default element gap (matches existing Create/Join) |
| lg | 24px | `p-6` | Card/form container padding (matches existing molecules) |
| xl | 32px | `px-8` | Page horizontal edge padding (matches Header) |
| 2xl | 48px | `py-12` | Vertical centering padding above/below login card |
| 3xl | 64px | `h-16` | Header height (existing; preserved on login page) |

Exceptions: Discord OAuth button minimum touch target height = 44px (override `p-2` → `py-3` on that button only to meet WCAG 2.5.5).

## Typography

All sizes use `font-sans` (Manrope). 3 sizes, 2 weights.

| Role | Size | Tailwind | Weight | Weight Class | Line Height |
|------|------|----------|--------|--------------|-------------|
| Body | 16px | `text-base` | 400 (regular) | `font-normal` | 1.5 |
| Label | 14px | `text-sm` | 400 (regular) | `font-normal` | 1.25 |
| Heading | 20px | `text-xl` | 600 (semibold) | `font-semibold` | 1.2 |

Notes:
- "DRAFT" wordmark in Header uses `text-xl font-bold` (existing; do not change).
- Toggle link ("New here? Create an account") uses Label (14px, 400).
- Page heading ("Sign in" / "Create account") uses Heading (20px, 600).
- Error message uses Label (14px, 400) in destructive color.

## Color

All tokens already defined in `src/routes/layout.css`. No new tokens needed for Phase 1.

| Role | CSS Token | Approximate Hex | Usage |
|------|-----------|----------------|-------|
| Dominant (60%) | `--color-bg-primary` / `bg-bg-primary` | slate-50 (`#f8fafc`) | Page background, header background |

Accent (`amber-500`) reserved for:
- Focus ring on the Discord button (`ring-amber-500` on `:focus-visible`)
- Hover underline on the sign-in/register toggle link

Discord Blurple (`#5865F2`) reserved for:
- The "Sign in with Discord" / "Sign up with Discord" button background only
- Hover: `#4752C4` (10% darker)
- Button text: white (`text-white`)

Text colors (existing tokens, already in use):
- Primary text: `text-text-primary` (slate-900)
- Secondary text: `text-text-secondary` (slate-700)
- Tertiary / helper text: `text-text-tertiary` (slate-500)

## Page Layout Contract

The `/login` route is a **full-page centered card**. The existing `Header` component is included at top.

**Primary visual focal point:** The Discord OAuth button (full-width Blurple on a neutral card). It is the only saturated element on the page — all surrounding elements (heading, toggle link, error) recede visually. Executor must not introduce competing saturated elements on this page.

```
┌─────────────────────────────────────────────────────────┐
│  Header (h-16, existing pattern)                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│          ┌──────────────────────────┐                   │
│          │  [Heading: Sign in /     │                   │
│          │   Create account]        │                   │
│          │                          │                   │
│          │  [Sign in with Discord]  │                   │
│          │   (Blurple button)       │                   │
│          │                          │                   │
│          │  [Toggle: New here?      │                   │
│          │   Create an account]     │                   │
│          │                          │                   │
│          │  [Error message, if any] │                   │
│          └──────────────────────────┘                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

Container classes (follow existing molecule pattern):
```

```

## Component Inventory

### LoginCard (new molecule: `src/lib/components/molecules/LoginCard.svelte`)

Internal state:
- `mode`: `$state('signin')` — `'signin'` | `'register'`

Elements:
1. **Heading** — `text-xl font-semibold text-text-primary` — "Sign in" or "Create account" (derived from `mode`)
2. **Discord OAuth button** — full-width, min-height 44px, Discord Blurple, white text, `rounded-md`, loading state disables + shows spinner
3. **Toggle link** — `text-sm text-text-tertiary` with inline anchor `text-text-secondary underline hover:text-amber-500` — "New here? Create an account" (signin mode) / "Already have an account? Sign in" (register mode)
4. **Error message** — conditional, `text-sm text-red-600`, renders only when `form?.error` is present

### `/login` route files
- `src/routes/login/+page.svelte` — imports `Header` + `LoginCard`
- `src/routes/login/+page.server.js` — `load` (redirect if already authed), `actions.signin` / `actions.register` delegating to Better Auth Discord OAuth

## Interaction States

### Discord OAuth button

| State | Visual |
|-------|--------|
| Default | Blurple bg (`#5865F2`), white text, `rounded-md` |
| Hover | Darker blurple (`#4752C4`), cursor pointer |
| Focus-visible | `ring-2 ring-amber-500 ring-offset-2` |
| Loading (after click) | `disabled`, reduced opacity (`opacity-75`), inline spinner (SVG, 16px, white, `animate-spin`) replacing button text |
| Disabled | `cursor-not-allowed opacity-50` |

### Mode toggle

| State | Visual |
|-------|--------|
| Default | `text-text-tertiary` label + `text-text-secondary underline` link |
| Hover | link color → `text-amber-500`, `no-underline` |

### Error message

| State | Visual |
|-------|--------|
| Present | Renders below button, `text-sm text-red-600`, no background |
| Absent | Element not rendered (no empty placeholder) |

## Accessibility Contract

- Discord button: `aria-label="Sign in with Discord"` (redundant with visible text but explicit for screen readers when in loading state)
- Loading state: `aria-busy="true"` on button during OAuth redirect
- Error message: `role="alert"` so screen readers announce on appearance
- Toggle link: standard `<a>` with `href` (mode swap via `$state`, not JS-only)
- Focus order: Heading → Discord button → Toggle link
