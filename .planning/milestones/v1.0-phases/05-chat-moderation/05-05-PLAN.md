  <how-to-verify>
    Start the dev server: `npm run dev`

    **Test 1 — CHAT-01: Team channel isolation**
    1. Open two browsers (or incognito windows). Sign in as two different users.
    2. User A joins Team A in a room. User B joins Team B.
    3. User A switches to "Team" tab in ChatPanel and sends a message.
    4. Verify: User A sees their message in Team A chat. User B's "Team" tab (Team B) does NOT show User A's message.
    Expected: Team A and Team B channels are completely isolated.

    **Test 2 — CHAT-02: Spectator channel isolation**
    1. Open a third window as a guest (unauthenticated user).
    2. Guest sees "Spectator" tab. Guest sends a message in spectator chat.
    3. Verify: Guest's message appears in spectator chat. Team A and Team B "Team" tabs do NOT show the message.
    4. Verify: Players on teams cannot see the "Spectator" tab (it should not be present for team players).
    Expected: Spectator messages stay in spectator channel only.

    **Test 3 — CHAT-03: Rate limiting (silent drop)**
    1. As any user, rapidly send 7+ messages in under 5 seconds.
    2. Verify: First 5 messages appear in the chat. Messages 6 and 7 do NOT appear.
    3. Verify: No error toast, no UI feedback — the drops are completely silent.
    Expected: 6th+ messages silently dropped, no user notification.

    **Test 4 — CHAT-04: Slur filter**
    1. As any user, send a message containing a word from src/lib/slur-list.json.
    2. Verify: The message does NOT appear for any participant (including the sender).
    3. Verify: No error message shown in UI (silent drop for slurs).
    4. Send a message over 500 characters.
    5. Verify: A validation error appears inline below the chat input: "Message too long. Max 500 characters."
    Expected: Slurs silently dropped; over-limit messages rejected with inline error.

    **Test 5 — HOST-04: Mute and unmute spectator**
    1. Open host's browser and a spectator's browser (unauthenticated guest or signed-in non-team user).
    2. Host sees the Spectators section. Spectator appears in the list.
    3. Host clicks "Mute" next to the spectator's name.
    4. Verify: Spectator's name now shows "(muted)" indicator in host's spectator list.
    5. Verify: "Mute" button changed to "Unmute" for that spectator.
    6. Spectator sends a message in spectator chat.
    7. Verify: The spectator's message does NOT appear in spectator chat for any viewer.
    8. Verify: The spectator sees no indication they are muted (no error, no feedback).
    9. Host clicks "Unmute". Verify "(muted)" indicator disappears. Spectator can send messages again.
    Expected: Silent mute enforced server-side; host UI reflects mute state; spectator unaware.

    **Test 6 — Layout verification**
    1. Verify ChatPanel appears as a right sidebar (280px) in both the lobby phase and the draft board phase.
    2. Verify no layout overflow or broken columns when ChatPanel is visible.
    3. Verify tab switching between "All" and "Team"/"Spectator" tabs works during lobby.
    4. Verify only the role-specific tab is shown during the drafting phase (no "All" tab).
    Expected: Consistent sidebar placement, correct tab visibility per phase.
