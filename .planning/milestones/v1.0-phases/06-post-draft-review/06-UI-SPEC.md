## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 12px (text-xs, uppercase) | 600 (semibold) | 1.2 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Display | 28px (text-2xl or text-3xl) | 600 (semibold) | 1.1 |

Notes:
- Body: class names used in the draft review item list (`text-sm text-text-primary`). Source: DraftSlot.svelte.
- Label: section group labels ("BANS", "PICKS") follow `text-xs font-semibold uppercase text-text-secondary`. Source: TeamDraftColumn.svelte.
- Heading: team column headings ("Team A", "Team B") follow `text-xl font-semibold text-text-primary`. Source: TeamDraftColumn.svelte.
- Display: the "Draft complete" review page headline uses `text-2xl font-semibold` — weight collapsed from 700 to 600 to satisfy the two-weight rule (regular 400 + semibold 600).

Two weights only: 400 (regular) for body text and 600 (semibold) for all non-body text (labels, headings, display title).

## Interaction Contract

### Copy Link
- Button label: "Copy link" (default state)
- On click: writes `fullUrl` to clipboard via `navigator.clipboard.writeText()`
- Success: "Copied" text appears inline (green-600) for 2000ms then disappears
- Failure: error text appears using existing `actionError` pattern (`text-sm text-red-600`)
- Source: CONTEXT.md D-07, existing `copyLink()` function in `+page.svelte`

### Back to Home
- Rendered as `<a href="/">` — not a `<button>` (navigation, not action)
- Label: "Back to home"
- No confirmation required
- Source: CONTEXT.md D-07, D-08

### No real-time subscription
- Review page is SSR-only. No lobby stream. No live updates. Snapshot loaded once in `+page.server.js`.
- Source: CONTEXT.md D-03

### Phase branch in +page.svelte
- Add `{:else if snapshot.phase === 'review'}` branch alongside `'drafting'`/`'cancelled'`
- Review branch renders `DraftReview` centered, full-width, no ChatPanel
- Source: CONTEXT.md code_context, +page.svelte existing branching

## Edge Case States

| Scenario | Display |
|----------|---------|
| Team has zero bans (edge: ban-less script) | Bans section is hidden (`{#if resolvedBansA.length > 0}`) |
| Team has zero picks (edge: cancelled before any picks) | Picks section is hidden similarly |
| Both teams empty (cancelled draft landing on review — unlikely per CONTEXT.md) | Show "Draft ended without picks or bans." below the team columns |
| `champion_id` null in action row | Skip that slot silently — do not render an empty DraftSlot in the review |
| Clipboard API unavailable | Show "Could not copy link" via `actionError` (existing pattern) |

Source: CONTEXT.md "Claude's Discretion" section (empty-team edge case handling).

## Accessibility Contract

- Team column headings use `<h2>` (matches existing TeamDraftColumn.svelte)
- "Back to home" is an `<a>` element, not a button — correct semantics for navigation
- "Copy link" is a `<button type="button">` — correct for clipboard action
- "Copied" confirmation uses `role="status"` or `aria-live="polite"` region to announce to screen readers
- DraftSlot `aria-label` already encodes action and team: "{championName} — {action} for Team {team}" — reuse unchanged
- Empty sections are hidden via `{#if}` rather than `aria-hidden` to avoid orphaned headings in the DOM
- Touch target minimum 44px maintained on all interactive buttons (Back to home, Copy link) — consistent with existing DraftSlot `min-h-[44px]` pattern
