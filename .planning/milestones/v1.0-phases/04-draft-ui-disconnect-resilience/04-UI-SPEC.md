## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding (py-0.5 px-2), inline chip gaps |
| sm | 8px | Compact element spacing, gap between champion card label and role |
| md | 16px | Default element spacing, grid gap between champion cards, card internal padding |
| lg | 24px | Section padding, gap between draft board columns and center grid |
| xl | 32px | Layout gaps between major draft board regions |
| 2xl | 48px | Major section breaks (top of draft board to page edge) |
| 3xl | 64px | Page-level vertical padding (not used in dense draft board) |

Exceptions:
- Touch targets for champion card click areas: minimum 44px tall (accessibility requirement, WCAG 2.5.5)
- Grace period overlay: full-screen (100vw × 100vh), positioned fixed
- Timer progress bar height: 4px (decorative, not a spacing token)

## Typography

All text uses Manrope (sans-serif). Two weights only.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Champion card name, pick/ban slot labels, spectator names, status messages |
| Label | 12px | 600 (semibold) | 1.2 | Section headers ("TEAM A", "BANS", "PICKS"), turn action label ("BAN", "PICK"), timer seconds label |
| Heading | 20px | 600 (semibold) | 1.2 | Active turn indicator ("Team A — Ban", grace overlay title) |

Source: Existing codebase uses `text-sm` (14px), `text-xs` (12px), `text-xl` (20px) with `font-medium`/`font-semibold`. Display size (48px) is newly introduced for the timer countdown per D-06.

## Color

Color tokens are defined in `src/routes/layout.css` using Tailwind v4 `@theme`. Hex values below resolve from Tailwind's slate palette.

| Role | Token | Approximate Hex | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `bg-bg-primary` / `--color-slate-50` | #f8fafc | Main page background, champion card backgrounds, pick/ban slot backgrounds |
| Secondary (30%) | `bg-bg-secondary` / `--color-slate-200` | #e2e8f0 | Draft board column backgrounds, slot borders, dividers, section backgrounds |
| Accent (10%) | `amber-500` (#f59e0b) / `amber-400` (#fbbf24) hover | #f59e0b | Active turn indicator highlight border, active captain's slot glow, "Submit pick/ban" confirm button |
| Urgency | `red-600` (#dc2626) | #dc2626 | Timer bar and countdown number when ≤10 seconds remaining; destructive text actions only |
| Destructive | `red-600` (#dc2626) | #dc2626 | "Cancel room" text button (inherited from LobbyHostBar pattern) |
| Positive | `green-600` (#16a34a) | #16a34a | "Reconnected — syncing…" banner, "Draft resumed" inline confirmation |

Accent reserved for:
- Active turn indicator border/ring (the slot or region showing whose turn it is)
- "Submit pick/ban" confirm button (primary CTA during a captain's turn)
- Selected champion card highlight ring (state: chosen but not yet submitted)

Disabled/unavailable champions: opacity-40 on the card, cursor-not-allowed, pointer-events-none. No color change — just reduced opacity.

Grace period overlay background: `bg-slate-900/70` (semi-transparent dark, covers full screen). Overlay card: `bg-bg-primary`. Source: D-07 (full-screen, semi-transparent).

## Interaction Contract

### Champion Selection (Captain's turn only)
1. Click champion card → card gains amber-500 ring (selected state). All other cards remain interactive.
2. A "Submit [Pick/Ban]" button appears below the grid (or fixed at bottom of ChampionGrid) — amber-500 background, semibold, "Submit Pick" or "Submit Ban" depending on turn action.
3. Click "Submit [Pick/Ban]" → RPC call fires (`pickBan`). Champion card transitions to disabled state (opacity-40). Turn advances server-side.
4. Non-captain participants: cards are pointer-events-none throughout. No selection state possible.
5. Double-click is not a shortcut — single click to select, button to confirm. This prevents accidental submissions.

### Timer
- Client derives seconds remaining: `Math.max(0, Math.ceil((turnEndsAt - Date.now()) / 1000))`
- Timer ticks via `setInterval` at 250ms (smoother than 1000ms for the progress bar, acceptable CPU cost for a draft room with <10 participants). Source: Claude's Discretion.
- At 0s, server auto-advances (DRAFT-04 already implemented). Client shows "0" and waits for server snapshot update.

### Pause Overlay (Disconnect)
- Appears immediately when server publishes disconnect event (captain departed).
- Blocks all interaction: overlay is `pointer-events-all`, z-index above draft board.
- Grace countdown renders in PauseOverlay using same TimerDisplay atom at Display size.
- On reconnect within grace: overlay dismisses, StatusBanner "Reconnected — syncing…" shows for 3s.
- On captain promotion: overlay dismisses, StatusBanner "[Name] is now captain for Team X. Draft resumes." shows for 3s.
- On cancellation: overlay replaced by full-page cancellation state (see Copywriting below).

### Reconnect Hydration
- StatusBanner "Reconnected — syncing…" (green-600) appears immediately on WS reconnect.
- Banner dismisses automatically after snapshot arrives (or after 3s maximum).
- Client re-renders from full server snapshot. No local state merging.

## Layout

```
+--------------------------------------------------+
| Header (Phases strip: Lobby | Drafting | Review) |
+--------------------------------------------------+
| TurnIndicator (full width, centered)             |
|  "Team A — Ban"   [active captain name]           |
|  [TimerDisplay: 23  ████████░░░ ]                |
+--------------------------------------------------+
| 3-col grid (max-w-6xl mx-auto px-4)             |
| TeamDraftColumn A | ChampionGrid | TeamDraftColumn B |
|  Bans (3 slots)   | 7×4 cards   | Bans (3 slots)  |
|  Picks (5 slots)  |             | Picks (5 slots) |
+--------------------------------------------------+
```

- Draft board replaces the lobby layout entirely when `phase === 'drafting'` (D-01).
- Conditional rendering inside existing `+page.svelte` via `{#if snapshot.phase === 'drafting'}` (D-01, code context).
- Max width for draft board: `max-w-6xl` (wider than lobby's `max-w-4xl` to accommodate 3 columns).
- On mobile (<md breakpoint): stacked layout — TurnIndicator → ChampionGrid → Team A column → Team B column.

## State Matrix

| UI State | Visual Treatment |
|----------|-----------------|
| Loading (stream undefined) | Existing "Loading room…" text pattern from +page.svelte |
| Lobby phase | Existing lobby layout (no change) |
| Drafting — spectator / non-captain | DraftBoard visible, cards pointer-events-none, no Submit button |
| Drafting — active captain | DraftBoard visible, cards interactive, amber ring on selection, Submit button visible |
| Drafting — inactive captain (other team's turn) | DraftBoard visible, cards pointer-events-none |
| Champion selected (not submitted) | Amber-500 ring on card, Submit button enabled |
| Timer ≤10s | Timer number + bar both red-600 |
| Timer at 0 | "0" displayed, Submit button disabled, awaiting server advance |
| Captain disconnected | PauseOverlay covers board |
| Grace expired — promotion | PauseOverlay dismisses, StatusBanner (green) for 3s |
| Grace expired — no team member | Full-page cancellation message (see Copywriting) |
| Reconnected | StatusBanner "Reconnected — syncing…" (green) for 3s |
| Draft complete | `snapshot.phase === 'review'` — Phases strip advances, review UI (Phase 6 scope) |
